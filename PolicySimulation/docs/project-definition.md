# Project Definition

## 1. Project Summary

본 프로젝트는 **입법·정책을 하나의 환경(Environment)** 으로 정의하고, 그 안에서 **행위자(Agent)가 학습하며 적응하는 과정**을 강화학습(Reinforcement Learning)으로 시뮬레이션하는 것을 목표로 한다.

정책이 변경될 때 발생하는 **우회 행위, 회피 전략, 대체 행동**은 기존 정책 설계 과정에서 충분히 정량화되지 못했다. 본 프로젝트는 이러한 간접 효과를 실험 가능한 모델로 추상화하여, 정책 설계자가 사전에 다양한 시나리오를 탐색할 수 있는 기반을 제공한다.

본 산출물은 정책 결정을 자동화하거나 대체하지 않으며, **정책 설계를 지원하는 시뮬레이션 도구**로서의 역할에 한정된다.

---

## 2. Problem Definition

현행 입법·정책 설계 과정의 주요 한계는 다음과 같다.

- 정책 시행 이전에 행위자의 **적응 전략 변화**를 예측하기 어려움
- 정책 효과 분석이 사후 평가 중심으로 이루어짐
- 우회 범죄 및 부작용에 대한 정량적 근거 부족

실제 사회에서 행위자는 고정된 규칙을 따르지 않고, **환경 변화에 반응하며 전략을 학습**한다. 그러나 대부분의 정책 분석은 이러한 학습 과정을 충분히 반영하지 못한다.

본 프로젝트는 이 문제를 **"정책 = 환경, 시민/행위자 = 학습 에이전트"**라는 관점으로 재정의한다.

---

## 3. Project Objective

본 프로젝트의 핵심 목표는 다음과 같다.

1. 정책을 수치적·구조적 파라미터를 가진 환경으로 모델링
2. 행위자의 우회·회피 전략을 강화학습 에이전트로 표현
3. 정책 변화에 따른 장·단기 결과 지표 변화 관찰
4. 대안 정책 시나리오 비교 실험 가능 구조 설계

궁극적으로는 **정책 설계 시 고려 가능한 선택지의 폭을 넓히는 것**을 목표로 한다.

---

## 4. Scope Definition

### In Scope

- 단순화된 정책 환경 정의
- 가상 또는 추상화된 행위자 모델
- 강화학습 기반 행동 전략 학습
- 정책 파라미터 변화에 따른 시뮬레이션 결과 분석

### Out of Scope

- 실제 개인 데이터 기반 예측
- 특정 범죄 유형에 대한 실증적 단정
- 법 집행 자동화 또는 정책 추천 시스템
- 실시간 정책 의사결정 지원

---

## 5. Modeling Concept

### 5.1 Environment (Policy)

정책 환경은 다음과 같은 요소로 구성된다.

- 규제 강도
- 처벌 수준
- 단속 확률
- 대안 제도의 존재 여부

이러한 요소는 **환경 상태(State) 또는 파라미터**로 표현된다.

### 5.2 Agent (Actors)

에이전트는 정책 환경 하에서 자신의 보상을 극대화하려는 행위자로 정의된다.

- 합법 행동
- 우회 행동
- 위험 행동
- 대체 전략 선택

에이전트는 반복적 상호작용을 통해 전략을 학습한다.

### 5.3 Reward Design

보상 함수는 다음 요소를 반영할 수 있다.

- 개인 효용
- 비용 및 위험
- 제재 발생 시 손실

정책 효과 평가는 **에이전트 보상 + 사회적 지표 변화**를 함께 관찰한다.

---

## 6. MVP Strategy

초기 MVP는 특정 법안이나 정책을 직접 다루지 않는다.

목표는 다음과 같다.

- 정책 시뮬레이션을 위한 최소 구조 검증
- 강화학습 적용 가능성 확인
- 결과 해석 프레임 정립

MVP 단계에서는 **범용성 확보와 구조적 타당성 검증**을 우선한다.

---

## 7. Evaluation Metrics

MVP 단계에서의 평가지표는 다음과 같다.

- 에이전트 전략 수렴 여부
- 정책 파라미터 변화에 따른 행동 분포 변화
- 시뮬레이션 반복 시 결과 안정성

정확한 예측력보다 **행동 변화 패턴의 해석 가능성**을 중시한다.

---

## 8. Risks & Considerations

- 과도한 단순화로 인한 현실 괴리
- 보상 함수 설계에 따른 편향
- 결과 오해석 가능성

이를 완화하기 위해 모델의 한계와 가정을 명확히 문서화한다.

---

## 9. Future Direction

- 정책 유형별 환경 템플릿 확장
- 다중 에이전트 환경 도입
- 웹 기반 시각화 대시보드 연계
- 정책·공공 도메인 전문가 피드백 반영

본 문서는 프로젝트 진행에 따라 지속적으로 업데이트된다.
