# MVP Roadmap

본 로드맵은 **지원사업 심사 기준에 맞춰 8주 내 검증 가능한 MVP를 구현**하기 위한 실행 계획이다.
핵심 목표는 완성된 서비스가 아니라, **정책 시뮬레이션 구조의 타당성과 실행 가능성 증명**이다.

---

## MVP Goal

- 정책을 환경(Environment)으로 모델링하는 구조 검증
- 강화학습 기반 행위자 적응 시뮬레이션 가능성 확인
- 정책 파라미터 변화에 따른 행동 패턴 비교 실험
- 결과 해석이 가능한 최소 시각화 산출물 확보

---

## Week 1 — Problem Framing & Scope Fixing

**목표**
정책 시뮬레이션 문제를 강화학습 환경으로 추상화하고 MVP 범위를 고정한다.

**주요 작업**

- 정책 환경의 핵심 파라미터 정의
- 행위자 유형 단순화(1~2종)
- 관찰할 결과 지표 선정
- In-scope / Out-of-scope 명문화

**산출물**

- `docs/project-definition.md` 1차 확정
- 환경·에이전트 개념 다이어그램 초안

---

## Week 2 — Environment Design

**목표**
강화학습 실험이 가능한 최소 환경(Environment)을 구현한다.

**주요 작업**

- 상태(State), 행동(Action), 전이 규칙 정의
- 정책 파라미터의 환경 반영 방식 설계
- 단순 확률 기반 단속·제재 로직 구현

**산출물**

- 환경 설계 문서 보완
- 초기 Environment 코드 스켈레톤

---

## Week 3 — Agent & Reward Modeling

**목표**
행위자의 적응 행동을 학습 가능한 구조로 구현한다.

**주요 작업**

- 에이전트 행동 공간 정의
- 개인 효용 기반 보상 함수 설계
- 제재·위험 요소 보상 반영

**산출물**

- Reward 설계 문서
- 단일 에이전트 학습 실험 코드

---

## Week 4 — Reinforcement Learning Experiment

**목표**
정책 환경 내에서 에이전트가 실제로 전략을 학습하는지 검증한다.

**주요 작업**

- RL 알고리즘 적용(PPO/DQN 등)
- 기본 정책 시나리오 학습
- 전략 수렴 여부 확인

**산출물**

- 학습 로그 및 결과 요약
- 정책 변화 전/후 행동 비교 데이터

---

## Week 5 — Policy Scenario Testing

**목표**
정책 파라미터 변화에 따른 행동 변화 패턴을 비교한다.

**주요 작업**

- 규제 강도/처벌 수준 시나리오 구성
- 동일 환경에서 정책 조합 실험
- 결과 안정성 검증

**산출물**

- 시나리오별 결과 비교 표
- 정책 민감도 분석 초안

---

## Week 6 — Result Interpretation & Visualization

**목표**
결과를 정책 설계 관점에서 해석 가능하게 정리한다.

**주요 작업**

- 행동 분포 변화 시각화
- 전략 전환 시점 분석
- 해석 가이드 문구 정리

**산출물**

- 결과 차트 이미지
- 해석 노트 문서화

---

## Week 7 — Demo & Documentation

**목표**
외부 공유 가능한 MVP 데모와 문서를 완성한다.

**주요 작업**

- 웹 기반 정적 데모 페이지 구성
- README 및 docs 정리
- GitHub Pages 배포

**산출물**

- Demo 웹페이지
- 최종 README

---

## Week 8 — Review & Feedback Loop

**목표**
지원사업 제출 및 후속 확장을 대비한 정리 단계.

**주요 작업**

- MVP 한계 및 가정 명문화
- 확장 로드맵 정리
- 피드백 반영 계획 수립

**산출물**

- MVP 회고 문서
- Next Phase 로드맵

---

## Evaluation Criteria (For Reviewers)

본 MVP는 다음 기준으로 평가된다.

- 강화학습 환경으로서의 정책 모델링 타당성
- 행위자 적응 행동의 관찰 가능성
- 정책 시나리오 비교의 해석 가능성
- 프로젝트 범위와 윤리적 경계의 명확성

---

본 로드맵은 프로젝트 진행 상황에 따라 조정될 수 있으며,
**정책·공공 문제에 적용 가능한 실험 프레임워크 구축**을 최우선 목표로 한다.
